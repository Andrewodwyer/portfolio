Overview

The Word is a dynamic and interactive platform designed to make discovering and sharing local events effortless. From live music and cultural festivals to workshops and community gatherings, the app serves as a central hub for event seekers and organisers.

Unlike mainstream event platforms that prioritise ticket sales over user engagement, The Word fosters a sense of community through interactive features like comments, likes, and event attendance tracking. Whether users are looking to attend or promote events, the app ensures a smooth, engaging experience tailored to their needs.

Identifying User Pain Points & Market Gaps

Challenges Faced by Users on Existing Platforms:

  • Scattered & Inconsistent Event Information: Many event websites have outdated or incomplete listings, making it difficult to find relevant and reliable event details.
  • Cluttered UI & Poor Navigation: Platforms like Facebook Events have a cluttered interface, making it hard to search for events by date, category, or location.
  • Complicated Event Creation & Management for Organisers: Many event platforms make it difficult for small organisers to promote their events effectively.
  • Mobile Accessibility Issues: Many local event websites are not optimised for mobile use, despite users frequently searching for events on the go.

I approached the design process with a user-first mindset, ensuring that both event attendees and organisers had an intuitive, engaging, and accessible experience.

Stage 1: Research & Strategy

  • User Research: Conducted surveys and interviews to understand how users currently find and manage events.
  • Competitor Analysis: Studied platforms like Eventbrite, Meetup, and Facebook Events to identify gaps in usability and engagement.

Stage 2: Information Architecture & Navigation Enhancements

Solution: Created a clear, logical structure to improve event discoverability.

  • Categories & Tags: Users can filter events by Music, Sport, Classes, Culture, and other tags for faster search.
  • Date & Location Filtering: A calendar feature allows users to view upcoming events by specific dates.
  • Intuitive Navigation: Clean, minimal interface with a consistent, easy-to-follow layout.

Stage 3: UX/UI Wireframing & Prototyping

Wireframing: Designed wireframes prioritising ease of browsing and event posting.

  • Created low-fidelity wireframes to map out key pages (Homepage, Event Details, User Dashboard, Event Creation).
  • Ensured an intuitive layout that allows for quick event searches and easy navigation.

High-Fidelity Prototyping: Developed a sleek, modern design focused on engagement.

  • Vibrant Colour Scheme: Purple & coral for energy, contrasted with light grey for readability.
  • Visual Hierarchy: Clear headings and bold CTAs guide users through the site seamlessly.
  • Mobile-first Design: Fully responsive across desktop, tablet, and mobile.

Stage 4: Community Engagement & Personalisation Features

  • User Profiles & Personalised Feeds: Registered users receive event recommendations based on their interests & past engagement.
  • Like & Comment System: Users can interact with event pages, discuss details, and share feedback.
  • Attendance Tracking: Users can mark themselves as attending and see who else is going.

Stage 5: Event Creation & Management for Organisers

  • CRUD Functionality: Organisers can Create, Read, Update, Delete their own events.
  • Real-Time Updates: Events can be edited post-publication (date changes, location updates, etc.).
  • Dashboard Analytics: Organisers can track event engagement (likes, comments, RSVPs).

Stage 6: Optimised Mobile Experience & Accessibility

  • Mobile-first Design: Events can be browsed and created effortlessly from a smartphone.
  • Fast, Responsive UI: No lag when searching or filtering events.
  • Accessible Colour & Typography Choices: Ensures readability for all users, including those with visual impairments.

Key Features

  • Event Discovery: Browse and search for events with advanced filtering options
  • User Profiles: Customisable profiles showcasing user's event history and interests
  • Interactive Event Pages: Detailed event information with image galleries and user reviews
  • Social Features: Comment on events, like posts, and follow other users
  • Event Creation: Allow users to create and promote their own events
  • Responsive Design: Seamless experience across desktop, tablet, and mobile devices
  • Admin Dashboard: Comprehensive backend for content management and user moderation

Design Considerations

Colour Palette

The colour scheme was carefully chosen to evoke energy and excitement while maintaining readability and accessibility:

  • Primary Colour: Vibrant Purple (#8A4FFF) - represents creativity and innovation
  • Secondary Colour: Coral Orange (#FF7F50) - adds warmth and encourages action
  • Background: Light Grey (#F8F9FA) - ensures content stands out and reduces eye strain
  • Text: Dark Grey (#333333) - provides strong contrast for improved readability
  • Accent: Teal (#20B2AA) - used sparingly for highlights and calls-to-action

Typography

  • Headings: Montserrat - Provides a modern, clean look
  • Body Text: Source Code Pro - Highly legible for long-form content and small screen sizes

Outcome & Impact

Final Thoughts

Developing The Word allowed me to tackle real-world UX challenges in event discovery and community building. By balancing functionality, engagement, and aesthetics, I created a platform that streamlines event discovery and creation for attendees and organisers.
